# Final Review Checklist — Before You Submit

Run this pass **before** submitting the idea/concept on the BAH 2026 portal.

## Content completeness (proposal)

- [ ] Problem Understanding — present and matches the official problem statement
- [ ] Proposed Solution — clear four-phase pipeline
- [ ] Methodology — data, model, graph, twin, evaluation
- [ ] System Architecture — diagram included and referenced
- [ ] Innovation / USP — **clearly highlighted** (connectivity-complete + resilience twin)
- [ ] Expected Impact — ties to NNRMS + named ministries
- [ ] Feasibility — technical, compute, team, time
- [ ] Technology Stack — listed
- [ ] Team Roles / Capabilities — 4 roles defined
- [ ] High-Level Implementation Plan — phased
- [ ] Timeline — aligned to hackathon dates
- [ ] Risk Analysis — risks + mitigations table

## Diagrams

- [ ] System architecture renders (Mermaid or ASCII)
- [ ] Pipeline flowchart renders
- [ ] Workflow / timeline diagram renders
- [ ] Dashboard mockup included (optional but valuable)

## Placeholders & accuracy

- [ ] Every `<…>` placeholder replaced (team name, member names, emails, institutions)
- [ ] Problem statement title matches the portal exactly
- [ ] All dates correct (1 Jul submit · 20 Jul shortlist · 21 Jul induction · 6–7 Aug finale)
- [ ] USP claims are consistent across proposal, README, and diagrams

## Quality

- [ ] Grammar and spelling checked
- [ ] Consistent formatting (headings, tables, capitalisation)
- [ ] No broken internal links between files
- [ ] Tables render correctly after any PDF/DOCX export

## Submission mechanics

- [ ] All required portal fields completed
- [ ] Files exported to required format (PDF/DOCX) if the portal needs uploads
- [ ] Uploaded files open correctly after upload
- [ ] Submitted **before** the 1 July deadline (not the final hour)
- [ ] Submission confirmation saved / screenshotted

## Strategic gut-check (does it win Game 1?)

- [ ] Does it answer the **full** problem (extraction → graph → criticality → simulation),
      not just segmentation?
- [ ] Is the **decision-support** angle obvious to a non-technical judge?
- [ ] Does it mirror the brief's language — *"from space insights to stronger cities"*?
- [ ] Is the USP something competitors **cannot** retro-fit in 30 hours?
